在 Linux 中，进程之间通过**信号**来通信。进程的信号就是预定义好的一个消息，进程能识别它并决定忽略还是作出反应。进程如何处理信号是由开发人员通过编程来决定的。

<center><b>Linux 进程信号</b></center>

| 信号 | 名称 | 描述                           |
| ---- | ---- | ------------------------------ |
| 1    | HUP  | 挂起                           |
| 2    | INT  | 中断                           |
| 3    | QUIT | 结束运行                       |
| 9    | KILL | 无条件终止                     |
| 11   | SEGV | 段错误                         |
| 15   | TERM | 尽可能终止                     |
| 17   | STOP | 无条件停止运行，但不终止       |
| 18   | TSTP | 停止或暂停，但继续在后台运行   |
| 19   | CONT | 在 STOP 或 TSTP 之后恢复执行。 |
