### 1. 使用 nano 打开文件

要在命令行下使用 `nano` 打开文件，可以这样：

```shell
$ nano myprog.c
```

如果启动 `nano` 的时候没有指定文件名，或者指定的文件不存在，`nano` 会开辟一段新的缓冲区进行编辑。如果你在命令行中指定了一个已有的文件，`nano` 会将该文件的全部内容读入缓冲区，以备编辑。

> 注意：在 `nano` 编辑器窗口的底部显示了各种命令以及简要的描述。这些命令是 `nano` 的控制命令。脱字符（`^`） 表示 <kbd>Ctrl</kbd> 键。因此， `^X` 表示的就是组合键 <kbd>Ctrl</kbd> + <kbd>X</kbd>。

> 提示：尽管 `nano` 控制命令在列出组合键的时候使用的是大写字母，但是在使用的时候，大小写字母都没有问题。

### 2. nano 控制命令

| 命令     | 描述                                          |
| -------- | --------------------------------------------- |
| Ctrl + C | 显示光标在文本编辑缓冲区中的位置              |
| Ctrl + G | 显示 nano 的主帮助窗口                        |
| Ctrl + J | 调整当前文本段落                              |
| Ctrl + K | 剪切文本行，并将其保存在剪切缓冲区            |
| Ctrl + O | 将当前文本编辑缓冲区的内容写入文件            |
| Ctrl + R | 将文件读入当前文本编辑缓冲区                  |
| Ctrl + T | 启动可用的拼写检查器                          |
| Ctrl + U | 将剪切缓冲区中的内容放入当前行                |
| Ctrl + V | 翻动到文本编辑缓冲区中的下一页内容            |
| Ctrl + W | 在文本编辑缓冲区中搜索单词或短语              |
| Ctrl + X | 关闭当前文本编辑缓冲区，退出 nano，返回 shell |
| Ctrl + Y | 翻动到文本编辑缓冲区中的上一页内容            |

> 说明：如果你输入 <kbd>Ctrl</kbd> + <kbd>T</kbd> 命令使用 nano 的拼写检查器的时候得到了错误消息 Spell checking failed: Error invoking 'Spell'，下面是一些解决方法。在你使用的 Linux 发行版中安装拼写检查器软件包 aspell。
>
> 如果 aspell 没能解决问题，以超级用户的身份编辑 `/etc/nanorc` 文件。找到文件的最后一行 `# set speller  "aspell -x -c"`，删除行首的字符 #。保存并退出。
