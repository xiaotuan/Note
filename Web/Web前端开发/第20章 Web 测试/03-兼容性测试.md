### 1. 平台兼容性测试

#### 1.1 基本原理

市场上有很多不同的操作系统类型，最常见的有 Windows、UNIX、Mac、Linux 等。Web 应用系统的最终用户究竟使用哪一种操作系统，取决于用户系统的配置，而 Web 开发者需要保障的是 Web 应用能够在所有用户眼里看起来都是完美的，因此需要查看页面在不同平台不同浏览器下的测试截图。

#### 1.2 测试目的

平台测试的主要目的是被测试项目能否在不同的操作系统平台上正常运行，包括同一操作系统平台的不同版本上正常运行。

#### 1.3 案例：使用 BrowserShots 完成平台兼容性测试

① 简介

BrowserShots 是一个免费的在线测试工具，能够模拟不同操作系统平台下不同浏览器对被测网页的访问，并提供网页访问截图。BrowserShots 的官方网站为：<http://browsershots.org>。

② 使用 BrowserShots 进行平台兼容性测试

在需要提交的表单框内输入需要测试的网址，如 "<http://www.hactcm.edu.cn>"，同时选择 Linux、Windows、Mac 下的 FireFox 38.0 浏览器进行测试，提交的网址会被加入到一个任务队列。

### 2. 浏览器兼容性测试

#### 2.1 基本原理

浏览器是 Web 客户端最核心的构件，来自不同厂商的浏览器对 HTML 标签（如表格间距、框架处理）、CSS 样式表（如编写规范）、JavaScript （网页元素名称、方法名称等）、ActiveX 控件、浏览器插件 plug-ins 和安全性等很多方面都有着不同程度的支持。另外，框架和层次结构风格在不同的浏览器中也有不同的显示，甚至根本不显示。

#### 2.2 测试目的

浏览器兼容性测试主要是在不同的浏览器环境下对 Web 应用的显示测试，使得 Web 应用的功能能够正确运行于广泛的浏览器软件上，保证用户不管采用何种浏览器，都能够正确地访问 Web 应用。

#### 2.3 案例：使用 Microsoft Expression Web 4 SuperPreview 进行浏览器兼容性测试

SuperPreview 是一个用来观察网页在不同浏览器中的显示效果，并帮助检测发生显示异常的原因的工具软件。可通过 <http://www.microsoft.com> 下载获得试用版本。

### 3. 分辨率兼容性测试

#### 3.1 基本原理

由于用户终端设备的不同，用户环境的分辨率也不相同，而分辨率对 Web 页面的展示效果影响很大，不同大小的分辨率会使页面排版、字体样式等的显示显著不同。

#### 3.2 测试目的

对 Web 系统的页面进行分辨率测试以确认用户使用操作系统的分辨率是否会影响 Web 应用的正常显示，并为页面的优化提供依据与建议，最终提升 Web 应用的整体表现，提高用户体验满意度。

#### 3.3 案例：使用 TestSize 进行分辨率测试

TestSize 是免费的在线测试工作，其可以模拟不同分辨率来查看 Web 的显示效果。TestSize 的官方网站为：<http://testsize.com>。
