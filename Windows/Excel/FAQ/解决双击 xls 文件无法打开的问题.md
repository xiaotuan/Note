 出现这种问题的原因有以下几种情况：

1. 如果这个故障不是一直出现的，那么有可能是用户的 `Excel` 在上次使用后（也可能是被其他程序调用后），没有正常退出，它作为一个句柄停留在系统进程中，成为了看不见而又确实在运行的 `Excel` 实例。这个实例有可能会干扰新实例的启动，而双击文件恰恰需要启动一个新的 `Excel` 实例，所以会没有反应。

   解决办法：先关闭电脑上的 `Excel`（如果正在运行的话），然后按 <kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>Esc</kbd> 组合键调出 `Windows` 任务管理器，找到并选定进程列表里面的 `Excel.exe` ，单击 "结束进程"。

2. 如果用户的 `Excel` 启动时会自动加载某些文件，而这些文件存在某些问题，也就会造成 `Excel` 无法正常启动。

   解决办法：删除 `Excel` 启动文件夹里面的所有文件。

   > 提示：启动文件夹位置，请参阅 [《Excel 启动时自动打开指定的工作簿》](../其他/Excel 启动时自动打开指定的工作簿.md) 文章中的 “方法二：利用 XLSTART 文件夹” 。

3. `*.xls` 文件与 `Excel` 程序的关联被破坏了。

   解决办法：重新建立文件关联。推荐 `Excel` 用户使用这种方法：单击 "开始" -> "运行"，输入命令 `Excel/regserver`，然后单击 "确定"。

4. 如果上述 3 种方法都试过还不行，最后还可以下面方式解决。

   启动 `Excel` ，单击菜单 "文件" -> "选项"，在 "选项" 对话框的 "高级" 选项卡中，将 "忽略使用动态数据交换(DDE)的其他应用程序(<u>O</u>)" 这个复选框前的勾去除即可。
