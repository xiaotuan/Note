### 1. 进程终止原因

终止进程也就是结束程序的执行，让它从内存中卸载。进程终止的原因可能有如下  4 种：

+ 主线程的入口函数返回。
+ 进程中一个线程调用了 `ExitProcess` 函数。
+ 此进程中的所有线程都结束了。
+ 其他进程中的一个线程调用了 `TerminateProcess` 函数。

### 2. 终止当前进程

要结束当前进程一般让主线程的入口函数返回。`ExitProcess` 是一个 API 函数，它会结束当前应用程序的执行，并设置它的退出代码，其用法如下所示：

```cpp
WINBASEAPI DECLSPEC_NORETURN VOID WINAPI ExitProcess(_In_ UINT uExitCode);
```

可以在程序的任何地方去调用 `ExitProcess`，强制当前程序的执行立即结束。但是在 C/C++ 应用程序应该避免直接调用这个函数，因为这会使 C/C++ 运行期库得不到通知，而没有机会去调用全局的或静态的 C++ 对象的析构函数。
