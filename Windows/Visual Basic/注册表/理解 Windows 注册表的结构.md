Windows 注册表是用于存储有关应用程序、用户和机器信息的仓库。它是存储配置数据（如用户偏好、数据库连接字符串、文件位置以及其他信息）的绝佳位置。

> 注意：不要污染注册表！要记住，注册表不是个人数据库。实际上，如果应用程序使用了数据库，将信息存储在数据库中通常会比较好。

注册表是一个层次结构——就像一棵树。树的顶级节点是预先定义号的——不能添加、修改或删除。

<center><b>Windows 注册表的顶级节点</b></center>

| 节点                | 说明                                                         |
| ------------------- | ------------------------------------------------------------ |
| HKEY_CLASSES_ROOT   | 包含将文件类型关联到程序的信息以及 COM 组件的配置数据        |
| HKEY_CURRENT_USER   | 包含当前登录到 Windows 的用户的配置信息                      |
| HKEY_LOCAL_MACHINE  | 包含与计算机相关的配置信息，不管用户是否登录                 |
| HKEY_USERS          | 包含计算机上所有用户的配置信息。用户登录时，HKEY_CURRENT_USER 就设置为 HKEY_USERS 中具体用户的别名 |
| HKEY_CURRENT_CONFIG | 包含本地计算机启动时使用的硬件配置文件                       |

键可以包含一个或多个值。值项有特定的数据类型，不过它们的数据类型可能与 Visual Basic 的数据类型不同。

<center><b>注册表值的常用数据类型</b></center>

| 属性          | 值                                                           |
| ------------- | ------------------------------------------------------------ |
| REG_SZ        | 这是字符串数据的主要类型，用于存储固定长度的字符串或其他短的文本值 |
| REG_EXPAND_SZ | 可扩展的字符串值，可以保存值在运行时才解析的系统变量         |
| REG_MULTI_SZ  | 以数组的格式保存多个文本字符串。每个字符串 "元素" 都以 null 字符结束 |
| REG_BINARY    | 用于村粗二进制数据                                           |
